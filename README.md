# DataFusionContest2026_Kiberpolka

Участникам необходимо решить задачу multi-label классификации для 41 финансового продукта клиентов банка на основе обезличенных данных заранее предоставленных признаков.

**Постановка задачи**

Клиенты банка владеют разными банковскими продуктами — различными видами счетов, карт и услуг. Одним из запросов бизнеса является желание ранжировать эти продукты по вероятности их открытия клиентами.

В данной задаче продуктов “на полке” 41 штука. В отличие от классических задач рекомендаций, бизнес интересуют вероятности открытия каждого из продуктов. Умея хорошо предсказывать эти вероятности, бизнес имел бы возможность гибко настраивать рекомендации.

В этом соревновании участникам предстоит работать с полностью анонимными и обфусцированными данными 1,000,0000 клиентов:

Описания банковских продуктов не передаются. Также не передаются и описания признаков. Для признаков передается только информация об их исходном типе (категорийные признаки cat_feature_i и числовые признаки num_feature_j). 
Признаков достаточно много: основной набор в 200 признаков, а также дополнительный набор с более 2,000 признаками.
В признаках много пропущенных значений, а также присутствуют выбросы.
750,000 клиентов имеют разметку и составляют тренировочные данные. Оставшиеся 250,000 клиентов составляют тестовые данные.

# РЕШЕНИЕ

**🚀 Multi-Target Customer Behavior Prediction (KiberPolka)**

Решение задачи прогнозирования 41 типа покупательского поведения на основе транзакционных данных и сложных признаков.

📊**Основные достижения**

**Leaderboard Score:** 0.8455+ (62 место на Лидерборде).

**Pipeline:** Разработан отказоустойчивый конвейер обучения на 5 фолдах с автоматическими чекпоинтами.

**Feature Engineering:** Обработка 750 000 признаков (Main + Extra features) с использованием Polars для высокой скорости.

🛠 **Технологический стек**

**Модели:** CatBoost, XGBoost(MixDepth), LightGBM(MixDepth), Neural Network (MLP), Random Forest.

**Валидация:** MultilabelStratifiedKFold (5 фолдов) — для сохранения баланса по всем 41 таргетам.

**Оптимизация:** Optuna для поиска весов ансамбля (Global & Target-wise blending).

**Инструменты:** Polars, Scikit-learn, PyTorch, Pandas, Numpy.

💡 **Ключевые фичи решения**

**Ensembling (Квинтет):** Объединение бустингов и нейросети для компенсации ошибок разных архитектур.

**Target-wise Optimization:** Индивидуальный подбор весов моделей для каждой из 41 колонки таргета.

**Memory Management:** Оптимизированный расчет Macro-AUC чанками (кусками) для работы со сверхбольшими массивами данных (750k+ строк) на ограниченном RAM.

**Rank Averaging:** Стабильное усреднение рангов для финального сглаживания предсказаний.


После серии экспериментов было установлено, что оптимальный баланс между точностью и обобщающей способностью дает ансамбль из 4-х моделей. Случайный лес (Random Forest) был исключен как вносящий избыточный шум.

### Состав ансамбля:
1. **XGBoost (MixDepth):** Основной локомотив, обеспечивающий максимальный AUC в большинстве категорий.
2. **CatBoost:** Эксперт по категориальным признакам, стабилизирующий общий результат.
3. **LightGBM:** Легкая и быстрая модель, дополняющая структуру бустингов.
4. **Neural Network (MLP):** Критически важный элемент, находящий нелинейные зависимости, которые пропускают деревья.

**Итог:** Поколоночная оптимизация весов для этого квартета позволила достичь пиковых значений AUC и минимизировать ошибки на редких таргетах.

# 📊 Анализ исходных данных (EDA)

**Таргет-анализ:** Описание дисбаланса (у нас 41 колонка, и некоторые покупки — большая редкость). Это объясняет, почему мы выбрали MultilabelStratifiedKFold.

**Группировка признаков:**

**Main Features:** Прямые агрегаты (суммы, средние, количество покупок).

**Extra Features:** Временные лаги, тренды (растет или падает активность клиента) и специфические "флаги" поведения.

**Важность признаков (Feature Importance):**

Перед обучением финального «Квартета» мы столкнулись с проблемой размерности: исходные данные содержали 2446 признаков. Использование всех признаков сразу приводило к переобучению и колоссальным затратам оперативной памяти.
Для решения этой задачи отбор признаков проходил разными методами и способами:

**🧬 Метод 1: «Лаборатория Качества»** - Автоматизированный отбор признаков (Feature Selection)

feature_selection - скрипт грубой очистки/отбора

**⚙️ Механика отбора:**

*Высокоскоростная обработка (Polars):*

Мы использовали библиотеку Polars для молниеносного объединения огромных таблиц (train_main + train_extra). Это позволило работать с 2500+ колонками без вылетов по памяти.

*Анализ на репрезентативной выборке:*

Чтобы ускорить процесс, отбор проводился на 50% случайной выборки данных. Этого достаточно, чтобы поймать основные статистические закономерности.

*CatBoost как судья:*

Была обучена модель CatBoostClassifier в режиме MultiLogloss (сразу на 41 таргет). Параметр rsm=0.1 (Random Subspace Method) заставлял модель при каждой итерации смотреть на случайные 10% признаков, что гарантировало проверку каждой из 2446 колонок.

*Жесткий порог важности:*

Мы отсекли всё лишнее. Признаки, чья важность (feature_importance) оказалась ниже 0.001, были признаны шумом и удалены.

**📊 Результат:**

До: 2446 признаков.

После: 792 наиболее значимых признаков (сохранены в selected_features.txt).

Профит: Этот шаг позволил сократить время обучения финальных моделей в 4 раза и повысить стабильность AUC на 2%.

**🧬 Метод 2:  «Высокоточный сканер»** - Иерархический отбор признаков (Multi-Level Selection)

ultra_feature_selection - скрипт высокоточного сканера, который разделил признаки на три «лиги» по их влиянию на результат.

Для исключения размерности данных была реализована система многоэтапной фильтрации. Из исходного массива в 2446 признаков выделены наиболее стабильные предикторы с помощью последовательных аналитических циклов.

**⚙️ Алгоритм «Тройного фильтра»**

Проведен глубокий аудит данных: обучена «оценочная модель» (CatBoost) на 2000 итерациях. Это позволило выявить как явные зависимости, так и тонкие паттерны, теряющиеся при стандартной тренировке. На основе полученных показателей важности (Feature Importance) сформировано три рабочих набора данных:

	Elite Core (0.05): Самые мощные предикторы.
	Optimal Balance (0.01): Сбалансированный набор для стабильного обучения.
	Broad Coverage (0.001): Максимальный охват сигналов для сложных нейросетевых архитектур.

**🧬 Метод 3: Формирование «Золотого Ядра»:** Метод пересечения (Intersection)

Вместо выбора одного из наборов применен метод математического консенсуса. Проанализировано пересечение четырех независимых выборок признаков, полученных в ходе экспериментов.

**📊 Аналитическая воронка отбора:**

	Elite 0.05: 399 признаков.
	Optimal 0.01: 988 признаков.
	Broad 0.001: 1484 признака.
	Model-Selected (500): 792 признака.

**🎯 Финальный результат:**
Оставлены только те 337 признаков, которые присутствовали в каждом из четырех списков.

**Данный подход позволил:**

Очистить сигнал: Оставить параметры, чья важность подтверждена.

Исключить шум: Убрать признаки, на которых модели были склонны к переобучению.

Достичь синергии: Создать единое информационное поле, в котором все 4 модели Квартета показали свой максимум.

**🧬 Метод 4: Турнир признаков:** Битва за финальный датасет

После того как было подготовлено несколько аналитических наборов данных (Elite, Optimal, Broad, Core и 500), возникла необходимость объективно определить победителя - наилучший набор. Для этого был реализован автоматизированный Турнир (Feature Set Tournament).

**Правила соревнований:**

*Репрезентативная выборка:*

Чтобы не тратить дни на обучение каждой версии, оценка проводилась на 20% случайной выборки данных. Это позволило сохранить статистическую значимость, увеличив скорость тестов в 5 раз.

*Равные условия:*

Все наборы признаков тестировались на одной и той же архитектуре CatBoostClassifier с идентичными гиперпараметрами и random_seed.

*Критерий победы:*

Главным судьей выступила метрика Macro-AUC. Победителем признавался тот файл, который обеспечивал максимальную обобщающую способность на валидационном фолде.

**🏁 Итоги турнира:**

Победу в честной борьбе одержал набор, полученный с помощью метода 1. Именно он показал наилучшее сочетание точности и устойчивости к переобучению, став официальным фундаментом для финального «Квартета».


**🧬 Метод 4:  «Метод иерархической фильтрации и консолидации признаков»**

**Первый этап метода — «Тонкая огранка данных».**

Если до этого мы работали со свойствами самих признаков, то здесь был совершен переход к инженерному анализу структуры таргета. Это позволило моделям «подсказывать» друг другу правильные ответы внутри логических групп товаров.

*💎 Глубокая очистка и иерархическая агрегация (Refinement)*

В середине соревновательного процесса был проведен повторный углубленный анализ, целью которого стало устранение скрытого шума и создание групповых признаков-подсказок. Это позволило перевести обучение на новый уровень понимания структуры спроса.

🛠 Ключевые нововведения этапа:

 * Санитарная очистка (Deduplication):

Проведен поиск и удаление полных дубликатов в пространстве признаков. Была решена проблема «информационного шума»: когда при идентичных характеристиках клиента фиксировались разные таргеты. Оставление только уникальных по поведению строк позволило моделям быстрее сходиться и не отвлекаться на противоречивые примеры.

 * Иерархическое проектирование (Group Context):

На основе анализа названий таргетов (например, target_1_1, target_1_2) была воссоздана логическая структура групп товаров.
Создано 10 новых мета-признаков (по числу основных групп).

Каждый признак отвечал на вопрос: «Есть ли вообще активность клиента в данной категории?» (pl.max_horizontal).

Эти «групповые подсказки» дали моделям контекст: если клиент в принципе покупает товары из «Группы 1», вероятность покупки конкретного товара 1_5 внутри этой группы резко возрастает.

Как результат было сделано формирование ULTIMATE-набора.

**Третий этап метода метода Финальное сжатие: Охота на «Красные квадраты»**

После объединения всех элитных признаков и иерархических групп был проведен финальный этап математической оптимизации — Squeeze-фильтрация. Целью было устранение избыточности (мультиколлинеарности) среди 500+ параметров.

⚙️ Технология «Быстрой охоты»:

*Высокоскоростная выборка:* Для расчета матрицы корреляций использовался чанк из 100 000 строк. Это обеспечило статистическую достоверность при колоссальной экономии времени и RAM.

*Детекция дубликатов (Порог 0.98):* Был применен жесткий фильтр корреляции Пирсона. При обнаружении признаков-близнецов (корреляция > 0.98), один из них безжалостно удалялся. Это предотвратило «раздувание» весов моделей и сделало их более устойчивыми.

*Гибридный подход (Polars + Pandas):* Расчеты велись в Pandas (через float32) для точности матричных операций, а финальная сборка и сохранение — в Polars для максимальной скорости записи .parquet.

**🏁 Результат сжатия:**

Этот шаг позволил сформировать SQUEEZED-набор — максимально плотный датасет, где каждый признак уникален и несет в себе новую информацию.
Эффект: Модели стали обучаться на 25-30% быстрее, а риск «перекоса» предсказаний из-за дублирующих признаков был сведен к нулю.

# «Архитектура ансамбля и инженерные вызовы».

**Этап «Промышленный конвейер обучения (Training Pipeline)»:**

**🏗️ Архитектура обучения**: Отказоустойчивый Multi-Model Pipeline

Для получения финального результата был разработан высоконагруженный скрипт обучения, объединяющий разнородные архитектуры в единый цикл кросс-валидации (MultilabelStratifiedKFold).

**🛠️ Ключевые инженерные решения:**

 * Система рестарта и чекпоинтов (Fault Tolerance):

Учитывая общую длительность обучения (десятки часов), внедрена система автоматического рестарта. Скрипт сканирует папку checkpoints и, в случае сбоя, возобновляет работу с того фолда или даже конкретной колонки (для LGBM/XGB), на которых прервался процесс.

 * Гетерогенные вычисления (GPU + CPU):
  
*Для оптимизации ресурсов использован комбинированный подход:*

*CatBoost (GPU):* Обучается в MultiLogloss режиме на видеокарте (использование VRAM ограничено 70% для стабильности), что позволяет обрабатывать 41 таргет одновременно.

*LightGBM (CPU) & XGBoost (GPU):* Обучаются поколоночно. Это позволило внедрить MixDepth — разную глубину деревьев для разных категорий товаров.

*Neural Network (Deep MLP):*
Реализована глубокая нейронная сеть на PyTorch с архитектурой 1024 → 512 → 256.

BatchNorm и Dropout: Для предотвращения переобучения на 700+ признаках.

GELU Activation: Использование современной функции активации для лучшего градиентного потока.

Multi-Output Layer: Финальный слой с 41 нейроном и Sigmoid выдает вероятности по всем категориям синхронно.

 * Управление памятью (RAM & VRAM Management):

После обучения каждого тяжелого блока (например, CatBoost) вызывается принудительная очистка: gc.collect() и torch.cuda.empty_cache(). Это позволило избежать утечек памяти при работе с массивами в 750 000 строк.
